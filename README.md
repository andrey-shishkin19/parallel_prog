# **Лабораторная работа №4: Параллельные вычисления с CUDA**

## Цель работы:

Модифицировать программу из лабораторной работы №1 (умножение матриц) для параллельного выполнения с использованием технологии CUDA на графическом процессоре (GPU). Провести серию экспериментов с разными конфигурациями блоков (4x4, 8x8, 16x16, 32x32) и разными размерами матриц (200, 400, 800, 1200, 1600, 2000).

## **Экспериментальная установка:**

### Язык: C/C++ с CUDA

### Конфигурации блоков: 4x4, 8x8, 16x16, 32x32

### Размеры матриц: 200, 400, 800, 1200, 1600, 2000

### Измеряемая величина: время выполнения (мс)

### Аппаратная платформа: NVIDIA GPU (Google Colab, Tesla T4)

### Среда выполнения: Google Colab

### Команды компиляции и запуска:

!nvcc matrix_final.cu -o matrix_final

!./matrix_final

## Выводы

### 1. Конфигурация 4x4 показывает наихудшую производительность среди всех GPU-конфигураций — это связано с недостаточным количеством потоков в блоке, что приводит к неполной загрузке вычислительных ядер GPU.

### 2. Конфигурации 8x8, 16x16 и 32x32 демонстрируют близкие результаты, однако 16x16 является оптимальным для больших матриц (1600, 2000).

### 3. Для маленьких матриц (200x200) все конфигурации, кроме 4x4, показывают одинаковое время (2 мс) — узким местом становятся накладные расходы на запуск ядра.

### 4. GPU-реализация на CUDA значительно превосходит последовательную CPU-реализацию. Ускорение достигает ~490 раз для матрицы 2000x2000 (122516 мс на CPU против 250 мс на GPU).
